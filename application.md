项目摘要：

现有学术文档解析工具依赖线性规则，难以处理多栏排版与悬浮脚注，导致细粒度元数据识别率低。本项目构建多智能体协同系统，采用自适应视觉切片技术，解决大模型难以识别微小标识的问题；利用思维链推理，解决不同来源数据相互冲突的问题；引入RAG记忆模块，将人工纠错转换成系统经验。最终打造高精度、低成本的科研成果管理方案，解决高校成果统计难题。



特色创新点（限100字以内，建议2-3点）：

1.提出自适应视觉切片技术，解决大模型对微小标识难以识别的问题。

2.利用思维链推理进行决策，解决多源异构数据相互冲突的问题。

3.集成RAG记忆模块，打造无需重训基座模型即可持续进化的系统。



一、系统定位与核心场景（新增）

1.1 系统定位

本项目不仅是一个文献抓取工具，而是面向高校科研管理的智能治理系统。它利用多智能体协作，解决传统 OCR 难以处理复杂版面（通讯作者角标、脚注邮箱、跨栏脚注）的痛点，并弥补 API 数据滞后或缺失带来的元数据不一致问题。

1.2 核心场景：批量科研绩效统计

输入：

1. 待查清单：包含 500+ 条 DOI 的 Excel 表格。

2. 基准名单：本校教师花名册（姓名、工号、学院）。

核心挑战：

o 多源异构：如何整合 Crossref（API）和 PDF/网页截图（视觉）的冲突数据？

o 模糊匹配：如何判断论文里的 Z.P. Liu 是否为名单里的 刘泽萍？

o 权益认定：精准提取通讯作者、共同一作、署名单位。

输出：符合学院要求的《科研成果统计报表》。

1. 输入 DOI，系统构造 https://doi.org/{doi} 并自动访问页面截图。

2. 从截图中识别作者姓名、单位、作者排序、共一与通讯作者信息。

3. 将作者信息与教师库进行匹配与仲裁，输出结构化统计结果。

项目拟解决的关键科学问题，研究内容、总体框架、重点难点、主要目标（此部分为重点阐述内容）：

&#x09;关键科学问题

1.1 复杂非结构化版面下的视觉-语义对齐与零样本认知机制问题

传统的文档解析依赖于大量标注数据的监督训练，但在面对学术论文中极度多变的版面（如多栏混排、脚注干扰、跨页图表）时，模型的泛化能力急剧下降。如何突破传统 OCR 对固定模板的依赖，构建面向任意复杂版面的零样本（Zero-shot）视觉感知机制？如何建立二维视觉特征与一维语义结构之间的映射关系，实现利用“空间位置”辅助“语义理解”？如何协同 VLM 与 LLM，解决通讯作者角标等细粒度实体在像素级感知与语义推理间的断层问题？

1.2 多源异构证据链的冲突博弈与概率性推理问题

学术元数据来源具有多源异构性（Crossref的API结构化数据与PDF的非结构化视觉数据），不同信源之间常存在数据冲突或时效性差异。如何建立基于证据权重的多源信息仲裁（Multi-source Arbitration）机制？如何构建基于思维链（CoT）的逻辑推理模型，通过“假设-验证-决断”路径实现自动化纠错？如何量化不同模态证据的置信度，有效抑制大模型在推理中的幻觉与事实一致性问题？

1.3 人机协同环境下的知识长效记忆与智能体自进化机制问题

当前智能体系统多为静态部署，缺乏在长期运行中利用人工反馈进行自我优化的能力，导致对长尾、异常样本的处理能力停滞不前。如何构建基于人机回环（Human-in-the-loop）的自进化机制？如何利用 RAG 技术将人工纠错转化为向量化经验，实现“少样本”动态提示注入？如何通过交互反馈驱动 Prompt 优化，实现系统在不重训基座模型前提下的性能持续涌现？

通过解决上述关键科学问题，本项目旨在构建面向复杂学术文档治理的多智能体协同（MAC-ADG）框架 。我们将致力于探索多模态大模型在垂直领域的认知边界，验证 “视觉感知-语义推理-多源仲裁”范式的有效性，推动高校科研情报处理从“人工规则堆叠”向“认知智能体自动化”跃迁，为科研成果数据资产化提供一套可扩展、自适应的智能解决方案。

二、研究内容

针对上述关键科学问题，我们设计了一个面向学术文档治理领域的MAC-ADG系统。本项目的研究内容将围绕“视觉感知机制”、“多源仲裁模型”、“长效进化架构”及“多智能体工程实现”四个维度展开，旨在打破非结构化文档与结构化情报之间的语义壁垒。

2.1 基于自适应视觉切片的零样本感知机制研究

本部分旨在解决“复杂非结构化版面下的视觉-语义对齐”难题，突破传统 OCR 线性扫描造成的“语义鸿沟”。

&#x09;高精度的零样本感知：视觉智能体

此部分的研究内容是理论的视觉落地。为了突破 LayoutLM 等模型需要大量标注数据训练的局限，本项目将构建基于大模型视觉编码（Visual Encoding）的零样本感知智能体。

1\. 视觉锚点重构：我们不训练专门的版面分析模型，而是利用 VLM的通用编码能力。智能体将 PDF 渲染为高分辨率图像，通过 OCR 提取文字坐标（BBox），并将这些坐标转化为 LLM 可理解的相对位置描述符（如 <Header>, <Left-Column>, <Footnote-Area>）。

2\. 视觉-语义对齐推理：在推理时，视觉智能体不仅仅输出文字，而是生成带有空间逻辑的描述。例如：“检测到‘Zhang San’右上角有符号‘\*’，且在页面底部检测到‘Corresponding Author’，因此推断Zhang San为通讯作者。”这种基于视觉逻辑的思维链，实现了对人类阅读过程的像素级复刻。

&#x09;面向细粒度感知的自适应视觉切片与坐标映射机制

针对 VLM 在处理整页 PDF 时存在的分辨率瓶颈（导致角标、脚注等微小标识丢失），我们设计了一套自适应多尺度视觉切片算法。

1\. 基于显著图的ROI动态锁定

系统首先利用轻量级 PDF 解析器（PyMuPDF）提取文档的逻辑版面树，构建基于文本密度和关键词（如 "Corresponding", "Email"）的显著图。不同于传统的全图扫描，本算法通过显著性阈值自适应锁定感兴趣区域（Region of Interest, ROI）。 例如，针对 IEEE 格式，系统会自动锁定首页底部 15% 的区域；针对 Elsevier 格式，则锁定标题下方的作者块区域。这种机制将 VLM 的注意力从“全图搜索”聚焦到“局部推理”，在不增加Token消耗的前提下，将有效像素密度提升数倍。

2\. 基于仿射变换的局部-全局坐标映射

为了解决切片及缩放导致的坐标空间不一致问题，我们在切片生成过程中构建了严格的坐标逆映射机制。

假设全局页面坐标为 P\_global=\[x,y]^T，局部切片坐标为 P\_local=\[x',y']^T。切片在全局页面的左上角偏移量为 T=\[x\_off,y\_off ]^T，为了适配 VLM 输入分辨率而进行的缩放因子为ɑ（当ɑ > 1时表示放大，ɑ < 1 时表示缩小）。

当 VLM 在切片坐标系中识别出通讯作者标记时，系统通过以下仿射变换方程将其还原为全局物理坐标：

P\_global=M⋅P\_local+T

其中变换矩阵 M 定义为逆缩放矩阵：

M=\[■(1/α\&0@0\&1/α)]

该机制不仅支持原分辨率裁剪（ɑ = 1），更能适配为了增强微小细节而进行的超分辨率切片（ɑ > 1）或为了适配模型窗口而进行的下采样（ɑ < 1）。这确保了系统在任意尺度的视觉输入下，仍能精确判断“星号”究竟是标记在第一作者（Zhang San）还是第二作者（Li Si）的右上角，实现了像素级的视觉-语义对齐。

2.2 基于贝叶斯思维链的多源证据冲突仲裁研究

本部分旨在解决“API 数据与 PDF 视觉证据不一致”时的决策难题，我们将借鉴贝叶斯推理范式，建立了用于描述多源异构证据可信度的概率仲裁模型。

我们假设最终输出的字段真值T是多源证据向量E、上下文语境C和模型先验知识 K的函数：

P(T|E) = Arbitrate (Eapi, Evis, Hhistory)

此公式的核心任务是验证并拟合在不同信源冲突条件下，最优解的判定规律。公式中各项的物理含义设计如下：

Eapi(结构化证据项）：来自 Crossref/OpenAlex 等 API 的数据。虽然结构化程度高，但往往存在滞后性或字段缺失（如缺少单位具体归属）。

Evis（视觉感知项）：来自视觉智能体对 PDF 截图的零样本解析。该项包含丰富的空间特征（Spatial Features），如作者名字旁边的角标位置。我们创新性地引入空间注意力权重 α，用于模拟人类对特定区域（如首页脚注区）的高关注度。

Hhistory(长效记忆项）：利用 RAG 技术检索历史人工修正案例。当当前样本与历史纠错样本的版面相似度 Sim(doc\_i,doc\_history)>θ 时，该项权重激增，指导模型复用历史经验。

2.3 基于对比学习的 RAG 长效记忆与自进化机制研究

传统的智能体系统通常存在灾难性遗忘问题，即人工审核员修正过的错误，在遇到类似样本时仍会重复发生。为了打破这一循环，本项目构建了基于 RAG的长效记忆模块，将“人工纠错动作”转化为系统的“隐式经验”，实现系统性能随使用时长增长的非线性涌现。

(1) 错误模式的向量化表征

我们不再简单存储文本日志，而是引入对比学习（Contrastive Learning）思想，构建由三元组 <Llayout, Eerror, Ccorrection> 构成的纠错案例库。 其中，Llayout 是通过视觉编码器提取的版面指纹（Layout Fingerprint）。系统通过对比损失函数，拉近具有相似排版结构（如特定期刊的双栏格式）的向量距离，推远截然不同版面的距离。这意味着系统能够“记住”特定期刊的视觉特征与常见解析陷阱的关联。 

(2) 上下文感知的动态提示注入

在推理阶段，系统引入了类比推理环节。侦察智能体在处理新任务前，会首先计算当前文档的版面指纹，并在向量库中检索 Top-K 最相似的历史纠错案例。 检索到的案例将被转化为 Few-Shot Context（少样本上下文）动态注入到当前的Prompt中。这种机制使得 MAC-ADG 系统能够像人类专家一样举一反三，在不重新训练基座模型的情况下，通过积累经验不断逼近更完美的解析准确率。

2.4 系统总体架构设计与工作流机制

为了将上述理论转化为可落地的工程系统，本项目研发了 MAC-ADG系统。该系统采用“控制-执行-记忆”解耦的模块化设计，通过有限状态机（FSM）实现多智能体的动态编排，确保在处理异构学术文档时兼顾效率与精度。

(1) MAC-ADG 系统分层架构设计

MAC-ADG 的系统架构由三个核心组件构成：

1\. 智能路由调度器 (The Orchestrator) 是系统的“躯干”，采用FSM进行全局状态控制。它实施成本优先策略：优先调度 API 工具，仅在数据置信度低于阈值时，才激活高算力的视觉工具。同时负责管理状态切换，确保任务不陷入死循环。

2\. 多智能体协作层集成了三个专职智能体，分别承担不同的认知任务：

侦察智能体：负责连接外部世界，熟练使用 Crossref、Unpaywall 等工具链，快速获取基础元数据。

视觉智能体：负责“看”懂文档，利用 DeepSeek-OCR 处理 PDF 截图，解决 API 数据缺失问题。

仲裁智能体：系统的“大脑”。它集成了一个基于思维链的推理框架，将 API 数据与视觉数据进行对抗性验证。它能识别出“API 数据版本过旧”或“PDF 解析乱码”等复杂情况，并做出最终裁决。

3\. 长效记忆中间件 (RAG Memory Middleware) 为了增强智能体的进化能力，我们设计了一个即插即用的记忆层：

纠错案例库：当人工审核员修正了 智能体 的错误后，修正前后的数据对会被向量化存储。

动态提示注入 (Dynamic Hint Injection)：在处理新任务时，系统会检索最相似的历史案例，将其作为Few-Shot Context注入到 智能体 的提示词中，使其具备“举一反三”的能力。

(2) 基于 FSM 的闭环迭代工作流程

MAC-ADG的工作流程是一个由FSM控制的闭环迭代过程。当系统接收到一个 DOI 或论文链接时，进入以下循环：

1\.	侦察阶段 (Scouting Stage)：侦察智能体首先介入。并发调用 API，若信息完整且置信度高，直接输出。

2\.	感知阶段 (Perception Stage)：若 API 数据缺失，系统流转至感知状态。视觉智能体启动无头浏览器抓取页面或下载 PDF，利用 VLM 技术对文档首页进行“视觉切片”，提取带坐标的富文本信息。

3\.	仲裁阶段 (Arbitration)： 对比 API 与视觉数据。若一致则奖励；若冲突则启动思维链推理，依据“原文优先”或历史记忆进行裁决。

4\.	进化阶段 (Evolution)： 若最终置信度仍低于阈值（如 95%），触发人工审核。修正结果回写至 RAG 记忆库，完成一次系统的“认知升级”。

三、重点难点

重难点1： 复杂非结构化版面下实现高精度视觉-语义对齐的零样本感知机制建模。

现有VLM通常将整页文档压缩至固定分辨率，导致页脚、角标等微小标识因像素信息丢失而无法识别。如何在不依赖标注数据的前提下，设计自适应视觉切片算法，在有限的Token窗口内动态锁定并增强关键区域（ROI），同时建立严格的可逆坐标映射机制，确保局部识别结果能无损映射回全局语义结构，是突破传统OCR与布局分析模型局限的首要技术挑战。

重难点2： 多源异构证据冲突消解中的概率推理与置信度量化建模。

当API结构化数据与PDF视觉解析结果不一致时，传统方法缺乏对证据源动态可信度的评估能力。如何构建一个基于贝叶斯理论的推理框架，将LLM的思维链（CoT）过程形式化为可计算的潜变量，并量化不同模态证据（如API时效性、视觉空间特征、历史记忆相似度）的权重，从而实现可解释、可追溯的自动化仲裁，是本项目在多源融合层面面临的核心理论难题。

重难点3： 面向长尾与异常版面的系统自适应进化机制构建。

当前智能体系统多为静态部署，难以利用运行时的人工反馈实现持续优化。如何设计一个高效的RAG记忆模块，不仅能存储纠错案例，更能通过混合检索机制（结合版面视觉特征与文本语义）精准匹配历史经验，并将具体案例抽象为可泛化的推理提示（Prompt），从而在不重新训练基座模型的前提下，使系统具备“举一反三”的类比学习能力，是实现系统长效性能提升的关键难点。



四、主要目标

本项目旨在构建一套高精度、低成本、可自进化的学术文档智能治理系统。我们首创自适应多尺度视觉切片技术，突破大模型对文档微小标识的零样本感知瓶颈；构建贝叶斯思维链推理框架，实现多源异构数据冲突的可解释消解；集成RAG长效记忆模块，打造无需重训基座模型即可持续优化的智能体系统。

最终，我们将交付MAC-ADG（多智能体协同学术文档治理）系统原型，在复杂学术文档元数据提取任务中实现整体识别准确率95%以上的目标；验证系统通过人工反馈实现性能持续提升的自进化机制；形成一套可扩展、自适应、具备长效学习能力的科研情报治理解决方案，推动高校科研成果管理从“人工规则堆叠”向“认知智能体自动化”范式跃迁。



拟采取的研究方案及可行性分析（包括研究方法、技术路线、实验手段、关键技术等说明）：

&#x09;关键技术

2.1 基于大模型视觉编码的零样本感知与视觉-语义对齐技术

针对传统解析方法依赖大量标注数据训练 LayoutLM 等专用模型的局限性，本项目提出了一种基于通用VLM的零样本感知范式。该技术不涉及底层神经网络的重新训练，而是通过视觉锚点重构与逻辑推理对齐，实现对任意复杂学术版面的“即插即用”式解析。

视觉锚点重构（Visual Anchor Reconstruction）为了解决 VLM 在处理纯像素输入时空间坐标感知的“漂移”问题，我们引入了OCR 辅助的锚点机制。系统首先获取文本的边界框（Bounding Box），将其映射为 VLM 可理解的语义化相对位置描述符 D\_pos。 对于页面上的任意文本实体 e，其位置描述符生成函数定义为：

D\_pos (e)=ϕ(x,y,w,h)→{⟨Header⟩,⟨Left-Col⟩,⟨Footer⟩,…}

这种符号化的空间锚点使智能体能够迅速定位“右上角”、“页脚”等关键语义区域，建立了像素空间到语义空间的映射桥梁。

在推理阶段，视觉智能体并非简单输出文本，而是生成包含空间逻辑验证的结构化描述。我们构建了基于视觉逻辑的验证方程：

IsCorresponding(A)=I(∃S\_symbol∈Near(A)∧∃T\_note∈Footer s.t.S\_symbol∼T\_note)

即：只有当作者 A 附近存在特殊符号 S\_symbol（如 \*），且在页脚区域存在解释该符号的文本 T\_note（如 "Corresponding Author"）时，才判定A为通讯作者。这种机制实现了对人类阅读过程的像素级复刻，有效解决了“张冠李戴”的语义鸿沟问题。



2.2 基于仿射变换的自适应多尺度视觉感知技术

自适应视觉切片（Adaptive Visual Slicing）旨在解决 VLM 在处理长文档和微小字号时的分辨率瓶颈。在通用视觉编码中，图像被压缩为固定数量的Token，导致高频细节（如角标、脚注）丢失。本项目提出了一种基于信息增益的动态切片模型，将文档解析建模为“全局粗搜”与“局部精读”的联合优化过程。

我们将全局页面坐标系定义为 Ω\_global，局部切片坐标系定义为 Ω\_local。为了实现像素级的视觉-语义对齐，我们构建了一个基于仿射变换（Affine Transformation）的坐标映射机制。其数学表达如下：

P\_global=M⋅P\_local+T

其中，P\_local=\[x',y']^T 为切片内的局部坐标，T=\[x\_off,y\_off ]^T 为切片相对于页面的位移向量。变换矩阵 M 定义为：

M=\[■(s\_x\&0@0\&s\_y )]

其中s\_x,s\_y为缩放因子（本项目中 s=1 以保持原分辨率）。

&#x20;

图 3-2 自适应多尺度视觉切片与坐标重映射原理图

借助轻量级布局分析网络，我们首先计算页面各区域的显著性得分（Saliency Score）。系统仅对显著性得分 S(region)>δ 的区域（如作者块、脚注区）生成高分辨率切片，从而在有限的 Token 预算下最大化有效视觉信息的覆盖率。这种机制从数学上保证了 智能体 能够“看清”任何细粒度的语义标识。



2.3 基于贝叶斯潜变量模型的证据融合与决策机制

针对多源异构数据的冲突问题，本项目采用贝叶斯决策理论（Bayesian Decision Theory）构建冲突消解模型。 不同于无法处理逻辑依赖关系的传统加权投票法，我们将最优真值的判定建模为最大后验概率（MAP）估计问题。

我们将最终的元数据真值定义为 T， API结构化证据为 E\_api，视觉非结构化证据为 E\_vis，上下文语境为 C。我们的目标是寻找最优的T^\*，使得在给定证据条件下后验概率最大：

T^\*=〖argmax〗\_T P(T|E\_api,E\_vis,C)

为了量化推理过程的可靠性，我们引入思维链（Chain-of-Thought）作为潜变量（Latent Variable）Z。根据概率论中的边缘化规则，将上述决策公式分解为：

P(T|E,C)∝∑\_Z▒P(T|Z,E)⋅P(Z|E,C)

其中：

P(Z|E,C) 代表推理生成模型：即仲裁智能体根据证据生成推理路径（如“因为 PDF 有星号，所以 API 数据过时”）的概率。

P(T|Z,E) 代表决策执行模型：即基于推理路径得出最终结论的确定性。

图 3-3 基于贝叶斯思维链的多源证据融合算法流程图

通过引入思维链 Z，模型不再是黑盒预测，而是显式地生成了“证据冲突—>逻辑推演—>最终裁决”的解释路径，有效解决了大模型在处理复杂冲突时的幻觉问题。

2.4 基于有限状态机的多智能体协同框架

MAC-ADG 是一个由FSM严格控制的多智能体协作系统。多智能体系统（MAS）通过将复杂的文档治理任务解耦，利用不同角色的智能体（侦察、感知、仲裁）协同工作，实现了超越单一模型的鲁棒性。

本项目将文档处理流程形式化建模为一个五元组M=⟨S,Σ,δ,s\_0,F⟩，其中 S 为状态集，Σ为事件集（如API\_Success, Low\_Confidence），δ 为状态转移函数, s\_0为初始状态：默认为侦察状态 (S\_scout)，即系统启动时优先尝试低成本 API 获取, F为终止状态集：即最终交付元数据真值的结束状态(S\_termination)。

我们的目标是寻找一条最优状态路径，以最小化计算成本并最大化解析准确率。

状态转移函数 δ:S×Σ→S 的核心控制流算法设计如下：

S\_next={■(S\_output,\&if S\_curr=S\_scout∧Conf(E\_api)>θ\_high@S\_vision,\&if S\_curr=S\_scout∧Conf(E\_api)≤θ\_high@S\_judge,\&if S\_curr=S\_vision@S\_human,\&if S\_curr=S\_judge∧Conf(T)<θ\_safe )┤

&#x20;

图 3-4 基于 FSM 的多智能体状态流转与决策机制图

不同的智能体按照分工并行执行任务，并由 FSM 控制器（Orchestrator）管理流转：

侦察智能体：负责低成本数据获取，产生 E\_api。

视觉智能体：负责高精度视觉补全，产生E\_vis。

仲裁智能体：执行贝叶斯推理，计算置信度 Conf(T)。

这种架构使得系统能够动态适应不同难度的文档，在“极速模式”与“深思模式”间自适应切换。



2.5 动态纠错记忆库与混合检索增强技术

为了解决智能体在长周期运行中的灾难性遗忘问题，并实现从“人工反馈”到“系统经验”的转化，本项目构建了包含混合检索器与结构化记忆缓冲区的中间件层。

1\. 混合检索评分机制 (Hybrid Retrieval Scoring)

针对学术版面的视觉特征，单纯的语义检索往往失效。我们构建了基于 RAG 的长效记忆模块，采用稀疏检索 (BM25) 与 稠密检索 (Dense Retrieval) 相结合的混合评分机制，以精准匹配具有相似排版特征的历史案例：

Score(Q,D)=α⋅〖Sim〗\_dense (v\_q,v\_d)+(1-α)⋅〖Sim〗\_bm25 (t\_q,t\_d)

其中, v\_q,v\_d为当前文档与历史案例的版面视觉向量（用于捕捉排版相似性）；t\_q,t\_d为两者的元数据文本特征（如期刊名称、出版社，用于捕捉领域相关性）。

2\. 结构化记忆切片 (Structured Memory Slicing)

我们将系统的记忆定义为一个动态缓冲区 M，它由若干功能独立的切片组成：

M={M\_layout,M\_error,M\_correction}

其中：

M\_layout：存储特定期刊（如 IEEE Trans.）的视觉版面特征向量；

M\_error：存储历史上 智能体 犯错的“负样本”，用于负面约束；

M\_correction：存储人工修正后的“正样本”，用于 Few-Shot 上下文注入。

&#x20;

图 3-5 动态纠错记忆库的数据回写与检索流程图

在每一轮人工审核发生后，系统执行“记忆回写”策略，将新的 ⟨M\_layout,M\_error,M\_correction⟩ 三元组向量化存入库中，从而在推理阶段实现对特定领域长尾错误的精准规避。

&#x09;实验手段

为确保本项目提出MAC-ADG系统各项创新技术的有效性与优越性，我们设计了以下实验，旨在定量评估其核心模块的性能及整体解决方案的效能。

1.自适应视觉切片技术的零样本感知能力验证实验

实验目的：验证本系统“自适应视觉切片技术”在突破大模型分辨率限制、实现对学术文档中微小标识（如通讯作者角标†、\*）零样本感知的有效性。

实验设计：选取包含复杂版式（多栏、悬浮脚注）的学术论文PDF样本集（N=500），分别采用以下三种方法提取通讯作者标记：

基线方法1（传统OCR+规则）：使用PaddleOCR进行全图识别，再通过规则匹配角标与脚注。

基线方法2（整图VLM）：将整页图像直接输入通用视觉大模型（Qwen-VL-Max），通过Prompt指令要求识别通讯作者。

本系统方法（自适应切片VLM）：采用本项目的自适应视觉切片技术，锁定作者栏与脚注区域进行高分辨率局部感知与坐标映射。

评价指标：计算各方法在通讯作者标记识别准确率、角标-作者关联正确率以及处理耗时上的差异。

预期结果：预期本系统通过局部高分辩率感知，能显著解决基线方法因压缩导致的细节丢失问题，准确率提升预计超过25%。

2.贝叶斯思维链框架在多源冲突消解中的效能实验

实验目的：评估“贝叶斯思维链框架”相比传统融合策略，在解决CrossrefAPI、网页HTML、PDF视觉证据等多源异构数据冲突时的决策鲁棒性。

实验设计：构建一个包含人工标注真实值的测试集（N=300），其中刻意引入了多源数据冲突场景（如API作者列表缺失、PDF与HTML期刊名不一致）。分别采用以下策略进行元数据字段仲裁：

&#x09;策略A（规则优先级）：设定固定的来源优先级（如Crossref>HTML>PDF）。

&#x09;策略B（多数投票）：简单统计各来源提供的候选值频率。

策略C（传统学习融合）：使用第一篇文章中所述的基于特征向量的轻量级融合网络。

策略D（本系统-贝叶斯思维链）：采用本项目提出的基于概率推理的贝叶斯思维链框架，显式建模证据可信度与推理路径。

评价指标：对比各策略输出结果与真实值之间的字段级F1分数、冲突场景下的正确决策率，并分析贝叶斯框架生成的推理链可解释性。

预期结果：预期本系统能通过显式建模证据可信度与推理路径，在高冲突场景下实现最优的裁决精度与可解释性。

3.RAG长效记忆模块对系统自进化影响的持续跟踪实验

实验目的：验证集成RAG记忆模块后，系统能否在不重训基座模型的情况下，通过积累人工纠错反馈实现性能的持续提升。

实验设计：采用时间序列分析。在系统运行中持续将人工纠错案例（版面指纹+修正逻辑）回写至RAG记忆库。定期在固定验证集（V0）与新版式测试集（V1）上评估性能。

评价指标：记录并绘制系统在V0验证集上的准确率变化曲线、在V1验证集上的泛化准确率，以及人工审核干预率随时间（或处理论文数量）的变化趋势。

预期结果：预期随着纠错案例的积累，系统准确率将呈非线性提升，并对新版式表现出更强的泛化能力，证明其具备持续进化特性。
